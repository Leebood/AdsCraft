# AdsCraft 广告分析引擎需求文档 v2.0

## 一、变更背景

### 问题
当前分析引擎使用固定基准值（如CTR>2.5%为"优秀"、CPA<$10为"优秀"）对指标打分，但存在根本缺陷：
- 用户截图无法确定投放地区、行业、产品类型
- 不同地区/行业的指标差异巨大（美国CPA可能$50算正常，东南亚$5才算正常）
- 固定基准必然失真，用户看到"CPA差"但实际在他的市场可能是优秀的

### 决策
**砍掉基准打分，改用指标关系分析 + 趋势对比**。这套逻辑全球通用，不依赖地区/行业信息。

## 二、核心分析逻辑（三层）

### 第1层：指标关系诊断（每次必做）
分析指标之间的逻辑关系，识别矛盾和异常组合。这是核心功能，单次截图即可使用。

**诊断规则表：**

| 组合条件 | 诊断结论 | 建议动作 |
|---------|---------|---------|
| CTR高 + 转化率低 | 素材吸引力强，但落地页/产品页有问题 | FIX: 优化落地页（加载速度、CTA按钮、产品描述），优先级：高 |
| CTR低 + 转化率高 | 素材不够吸引人，但看到产品的人会买 | SCALE: 增加预算扩大曝光，同时优化素材提升CTR |
| CPA高 + ROAS低 | 花钱多但产出少，整体效率差 | STOP: 暂停当前投放，重新定义受众和素材 |
| CPC高 + CTR低 | 单次点击贵且点击率低 | FIX: 素材和受众匹配度差，优先更换素材或调整受众定位 |
| 频次高 + CTR下降 | 受众疲劳，素材被看腻了 | FIX: 立即更换素材或扩展受众池 |
| CPA低 + ROAS高 | 整体表现健康 | SCALE: 可加大预算放量，保持当前策略 |
| 所有指标均无明显异常 | 表现稳定 | 继续观察，建议上传下一批数据对比趋势 |

**指标高低的判断标准（仅用于组合诊断，不用于单独打分）：**
- CTR：>1.5%视为高，<1%视为低
- 转化率：>2%视为高，<1%视为低  
- CPA：结合ROAS判断，ROAS<2视为高
- ROAS：>3视为高，<2视为低
- CPC：结合CTR判断
- 频次：>3视为高

### 第2层：趋势对比（历史数据≥2条时启用）
将本次数据与用户的历史快照对比，展示变化趋势。

**逻辑：**
1. 从ad_snapshots表查询同一用户的所有历史记录，按created_at排序
2. 取最近一条与当前数据对比
3. 计算每个指标的变化率：`(当前值 - 上次值) / 上次值 × 100%`
4. 变化幅度≥20%标记为显著变化（↑上升为改善用绿色，↑上升为恶化用红色）

**输出格式：**

## 四、行动建议生成规则

### 动作类型
- **STOP**：立即暂停。适用于CPA高+ROAS低的组合，当前投放策略整体无效
- **FIX**：修复优化。适用于有明确矛盾点的组合（如CTR高+转化低），指向具体问题
- **SCALE**：放量扩大。适用于整体表现健康的组合，鼓励加大投入

### 优先级排序
1. STOP类优先级最高（止血）
2. FIX类按影响面排序（影响钱最多的排前面）
3. SCALE类优先级最低

### 每条建议必须包含
- 动作类型（STOP/FIX/SCALE）
- 针对哪个指标
- 具体要做什么（不能说"优化素材"这种空话，要说"更换视频前3秒hook"这种具体动作）
- 证据引用（引用具体数字，如"CTR 1.6%说明..."）
